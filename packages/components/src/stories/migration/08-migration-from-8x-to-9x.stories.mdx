# Migration from 8.x to 9.x

The following workflow walks through the steps of migrating.
Note that the actual steps required for your project may vary, and these steps should be treated as general guidance rather than strict instructions.

## Tabs, Accordion and Popover

The components tabs, accordion and popover can be use with two-way binding (ngModel or v-model).

```html
<!-- angular -->
value = 'tab-a'
<bal-tabs [(ngModel)]="value">...</bal-tabs>

<!-- vue -->
value = ref('tab-a')
<bal-tabs v-model="value">...</bal-tabs>
...
```

## Dropdown / Popover

bal-dropdown is renamed to bal-popover.

## Hint

Renamed the methods open and close to present and dismiss.

## Form fields

All the fields are set as expanded. All the expanded attributes in the form section can be removed.

## Modal

The bal-modal-footer and bal-modal-actions are removed due the adjusting the scroll behaviour.

```html
<!-- - old -->
<bal-modal>
  <bal-modal-header>Modal Title</bal-modal-header>
  <bal-modal-body>
    <p>
      Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore
      magna aliqua.
    </p>
  </bal-modal-body>
  <bal-modal-footer>
    <bal-modal-actions>
      <bal-button color="link">Cancel</bal-button>
      <bal-button color="primary">Okay</bal-button>
    </bal-modal-actions>
  </bal-modal-footer>
</bal-modal>

<!-- + new -->
<bal-modal>
  <bal-modal-header>Modal Title</bal-modal-header>
  <bal-modal-body>
    <p>
      Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore
      magna aliqua.
    </p>
    <bal-button-group position="right">
      <bal-button color="link">Cancel</bal-button>
      <bal-button color="primary">Okay</bal-button>
    </bal-button-group>
  </bal-modal-body>
</bal-modal>
...
```
