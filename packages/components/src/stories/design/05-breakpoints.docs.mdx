# Breakpoints / Viewports

The library supports following four breakpoints.

[Go to the Bulma documentation](https://bulma.io/documentation/overview/responsiveness/#breakpoints)

| Device     | Starts at screen width |
| ---------- | ---------------------- |
| Mobile     | 0px                    |
| Tablet     | 768px                  |
| Desktop    | 1024px                 |
| Widescreen | 1440px                 |
| Fullhd     | 1920px                 |

## JavaScript utility

`isPlatform` returns a boolean if the Design System is running in the given **device**.

```typescript
import { isPlatform } from '@baloise/design-system-next-components'

isPlatform(window, 'mobile')
isPlatform(window, 'tablet')
isPlatform(window, 'desktop')
isPlatform(window, 'widescreen')
isPlatform(window, 'fullhd')
```

## Sass mixins

There are nine responsive mixins, which can be easily used in your sass files.

Use the scss mixins in your application the following:

```css
@import 'node_modules/@baloise/design-system-next-components/src/styles/global.utilities';

.bal-element {
  width: 100%;
}

@include tablet() {
  .bal-element {
    width: 380px;
  }
}

@include desktop() {
  .bal-element {
    width: 570px;
  }
}
```

The file `global.utilities` provides all defined scss variables like [colors](?path=/docs/design-colors--page) and also the mixins.
